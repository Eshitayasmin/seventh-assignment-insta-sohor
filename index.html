Question: How does javascript work?

Answer:

javascript is a high-level, interpreted language. javascript runs with v8 engine. V8 engine compiles javascript code into machine code by JIT(Just In Time) compiler. When v8 engine compiles javascript code, the parser genarates an AST(abstract syntax tree). Abstract syntax tree is a tree representation of the javascript code. V8 engine uses a interpreter called ignition. Ignition takes AST and genarates bytecode. Turbofan is a v8 engine compiler. It takes bytecode from ignition. Then it compiles bytecode into optimized machine code.

Question: What does event loop do?

Answer:

javascript has a run time modal based on event loop. It is a combination of synchronous and asynchronous events. Synchronous events are performed one at a time and only when one is completed. The call stack is a mechanism that helps the JavaScript interpreter to keep track of the functions. When we call a function its pushed to the top of the call stack. But when fuction returns its removed from call stack. Call Stack is based on the Last In First Out (LIFO) principle. It means That function called at the end will pop out first. But when the function contains an asynchronous event(setTimeOut, setInterval, fetch, async e.t.c) it will show after all synchronus event, that's called call back queue.

Question: Difference between local storage and session storage

Answer:

Local storage and Session storage are almost the same. They are both store name value pair data for a particular domain. Major difference is..

1. Local storage store data parmanetly but Session storage stored temporarily.

2. Local storage have 5-10MB storage space but session storage have 5MB storage space.

3. Local storage data has to be deleted manually but if close the browser session storage data automatically clears.

4. Local storage works for any tab but session storage works for specific tab.
